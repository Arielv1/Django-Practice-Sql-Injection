Detection Tools for SQL Injections

In this page i will give you a brief information on how to use sqlmap and Vega 2 free tools that can help you detect vulnerabilities of sql injections

lets start with sqlamp

sqlmap is one of the most popular open-source testing tools to preform sql injection.
so in order to use sqlmap you need to download it :).
and the command to test vulnerabilities is:
sqlmap -u URL
The URL will work if you can pass data in the URL for example:
http://www.example.com/problems/1?id=22 this will pass the variable id with value of 22
as you can see variable come after the question mark ?
In the process, sqlmap can ask various questions and you need to answer y (i.e. Yes) or n (i.e. No).
Example of an basic run:
sqlmap -u http://www.example.com/problems/1?id=22



as you can see we may have vulnerabilities that can be exploited with the parameter id.
so we can learn that the website is vulnerable to sql injection.
also in this result we can see that the database used is MySql so sqlmap ask us if we want to check on more databases.
This is basic checks that you can do using sqlmap i will show you more things and commands that you can do:
there is alot more then only sql injections on sqlmap :)
To get a list of databases, use the --dbs option.
For example:
sqlmap -u http://www.example.com/fproblems/1?id=22 --dbs
lets say two databases were found:
[*] information_schema
[*] main_example

If you want to get the list of tables in the main_example database. To do this, use the --tables option.
you need to specify the database after the -D option:
sqlmap -u http://www.example.com/problems/1?id=22 -D main_example --tables
To print the entire contents, use the --dump option. It can be specified along with the database,
and then the entire database will be dumped, or you can restrict the data to one table or even one column.
for example:
sqlmap -u http://www.example.com/problems/1?id=22 -D main_example -T users --dump

Vega

Vega is also free and open-source but unlike sqlmap vega have graphical user interface. Vega is Vulnerability scanner and offer in 1 click alot of information. after scanning you will get alerts grouped by types High , Medium , Low , Info you need to remember not every alert is an Vulnerability it might be false-positive alert , which means it is not vulnerability, but it is recommended to look on every alert to know why vega put it as alert. it is really easy to use and not needed a guide. but i will show you how to do a basic search. After installing vega , click on "Start new Scan" , Window will be opened then enter a URL and click on Next

Vega has dozens of modules designed to find a wide range of common web server vulnerabilities such as SQL, XSS, and XML injection vulnerabilities. If you want to enable all of the scanner modules, make sure "Injection Modules" is checked. If not, expand "Injection Modules" and select the ones you want, and deselect the ones you don't.

You could click "Next" a few times to adjust Authentication Options or Parameters,else click "Finish" to start scanning the website.

When the scan is complete, Vega will show you a summary of the alerts. Looking like this:

Vega does a great job explaining what each alert means, their impact on our websites, and ways to remedy the vulnerabilities. It also includes useful references that may help us better understand how to deal with the vulnerabilities.

Clicking on one of the alerts will display much more useful information. for an example:
