Detection Tools for SQL Injections

In this page we'll provide brief information on how to use sqlmap and Vega 2, free tools that helps detecting vulnerabilities of SQL injections

Sqlmap

Sqlmap is a free, open source software, that is used to detect and exploit database vulnerabilities and provides options for injecting malicious codes into them.
You can download sqlmap from the here
Once downloaded, in order to launch the application on your device open command line and run:
sqlmap -u URL
When writing the URL address, it's possible to pass data, for example:
http://www.example.com/problems/1?id=22
Here we're passing the optional variable id with the value of 22
When running Sqlmap commands, you'll run into some prompts asking you how to process, for example when running:
http://www.example.com/problems/1?id=22
You'll receive the following image:


Based on the results we that the website has several vulnerabilities that are related to the id variable, from here we infer that it also contains SQL injection vulnerabilities.
In addition we've learned the specific type of database user, MySQL

Let's see some more tricks of sqlmap:
1. To get a list of databases, use the --dbs option.
sqlmap -u http://www.example.com/fproblems/1?id=22 --dbs
2. Lest assume we know of two databases:
[*] information_schema
[*] main_example
To get a list of tables in main_example we'll add a couple of keywords: --D and --table
sqlmap -u http://www.example.com/problems/1?id=22 -D main_example --tables
To print the entire contents, use the --dump option. It can be specified along with the database, and then the entire database will be dumped, or you can restrict the data to one table or even one column.
sqlmap -u http://www.example.com/problems/1?id=22 -D main_example -T users --dump
Checkout this sql user guide for more commands and tutorials

Vega

Vega is also free and open-source, however, unlike sqlmap Vega has graphical user interface.
Vega is a vulnerability scanner and provides in a single click a lot of information.
After the scan you'll get alerts grouped by types: High ,Medium ,Low and Info.
It's important to understand that the alerts doesn't equate to vulnerabilities, it might be false-positive alert, it is recommended to look at every alert to understand why Vega showed them.
Let's dive in for some examples:
After installing Vega , click on Start new Scan,a window will be opened with a URL input, after providing a URL click Next
Vega has dozens of modules designed to find a wide range of common web server vulnerabilities such as SQL, XSS, and XML injection vulnerabilities.
If you want to enable all of the scanner modules, make sure "Injection Modules" is checked.
If not, expand "Injection Modules" and select the ones you want, and deselect the ones you don't.
You could click Next a few times to adjust Authentication Options or Parameters,else click Finish to start scanning the website.

When the scan is complete, Vega will show you a summary of the alerts. Looking like this:

Clicking on one of the alerts will display much more useful information.
